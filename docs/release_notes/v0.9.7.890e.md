# Release v0.9.7.890e — HSS backward-corruption fix

**Date:** 2026-07-18
**Tag:** `v0.9.7.890e`
**Type:** Patch release (fixes the HSS training backward pass)

## Summary

`v0.9.7.890e` is a correctness patch for the Hierarchical State Space (HSS)
training path. It does not change the public API. Two bugs that made
`test_train_integration` diverge (loss increase instead of decrease) are fixed.

## What changed

### 1. Layer-norm gamma/beta gradient pointer dereference (`kernel/autodiff/ops.c`)

In `backward_layer_norm`, the gain/shift parameters were read as:

```c
float* gd = (float*)c->gamma->data;   /* WRONG: c->gamma->data is a tensor */
float* bd = c->beta ? (float*)c->beta->data : NULL;
```

`c->gamma` is a `SNEPPXVariable*`; `c->gamma->data` is its `SNEPPXTensor*`.
Casting the **tensor pointer** to `float*` read the tensor struct's bytes as
floats, yielding garbage gains. The fix reads the actual float buffer:

```c
float* gd = c->gamma ? (float*)c->gamma->data->data : NULL;
float* bd = c->beta  ? (float*)c->beta->data->data  : NULL;
```

Consequence before the fix: the layer-norm input gradient (`gx`) was scaled by
garbage gains, producing huge gradients that propagated through the whole graph
and exploded the Adam parameter updates for `x_proj` / `x_proj_bias`.

### 2. Missing `<math.h>` — `sqrt` implicitly declared (`kernel/autodiff/ops.c`)

`ops.c` did not include `<math.h>`, so `sqrt` was implicitly declared (MSVC
assumes `int sqrt()`). The call returned a constant `-256.0` for every input,
so `inv_std = 1.0 / sqrt(var + eps)` was wrong (≈ `-0.0039` instead of ≈ `14.8`),
corrupting the input gradient of **every** layer-norm in the graph.

Fix: added `#include <math.h>`.

### 3. Diagnostics removed

All debug instrumentation added during diagnosis (`[LN*]`, `[MSE*]`, `[CAT*]`,
`[TR*]`, `[RGTS]`, `[MSENG]`, `[TP-*]` prints, the `g_dbg_*` globals, and the
`_CRTDBG` flags in the integration test) has been removed.

## Result

`tests/unit/test_train_integration` now converges deterministically:

```
init_loss = 0.234858
final_loss = 0.015788   (ratio 0.067 — >93% reduction over 10 Adam steps)
```

Verified 15/15 Release builds.

## Upgrade notes

No code or API changes. Consumers rebuilding against `v0.9.7.890e` simply get a
correct HSS backward pass. See [`../migration/v0.9.7.890e.md`](./migration/v0.9.7.890e.md).

## Full feature set (carried from v0.9.7.890)

- Real format readers/writers: safetensors, NumPy (`.npy`/`.npz`), PyTorch
  (`.pth`), ONNX.
- Real tensor kernel: N-D matmul + gradient-descent optimizer on the autodiff
  tape (no stub shadows).
- Seven neural architectures: Transformer, ViT, GCN, RNN, GAN, Diffusion, RL,
  plus ARC / SER / HSS / NPE / FM.
- Hardened S0–S9 security.
- Opt-in backends (Vulkan, TPU, HTTP, ZK, Metal, oneAPI) performing real
  reference computation, OFF by default.
