# Migration guide: v0.9.7.890 → v0.9.7.890e

**Applies to:** Anyone consuming SNEPPX-Alg who builds/trains the HSS model.

## TL;DR

`v0.9.7.890e` is a drop-in correctness patch. **No API, ABI, or build-flag
changes.** Rebuild and you get a working HSS backward pass.

## What you must do

Nothing — just pull the tag and rebuild:

```powershell
git fetch --tags
git checkout v0.9.7.890e
cmake -B build -DCMAKE_BUILD_TYPE=Release -DSNEPPX_BUILD_TESTS=ON
cmake --build build --config Release
cd build && ctest -C Release --output-on-failure
```

The integration test `test_train_integration` is the regression guard for this
fix.

## What changed under the hood

| Area | Before (v0.9.7.890) | After (v0.9.7.890e) |
|------|---------------------|----------------------|
| `backward_layer_norm` gain read | cast tensor pointer to `float*` (garbage) | read `tensor->data` buffer (correct) |
| `sqrt` in autodiff | implicitly declared, returned `-256.0` | `#include <math.h>` added, correct value |
| HSS training loss | increased / diverged | decreases >90% over 10 steps |
| Debug prints | present in `ops.c`/`tape.c`/test | removed |

## Behavior changes you may notice

- **Training now actually learns.** If you previously saw HSS training loss
  increase or NaN/explode, that was the bug. After upgrading, loss should
  decrease monotonically (modulo LR/step-count choices).
- **Determinism.** With a fixed seed and `SNEPPX_OPTIMIZER_ADAM`, results are
  reproducible run-to-run (verified 15/15 Release runs).
- **No output-format change.** Saved weights, checkpoints, and log lines are
  unchanged except for the removal of the temporary debug prints.

## If you forked `kernel/autodiff/ops.c`

If you maintain a local patch to `ops.c`, port these two changes:

1. In `backward_layer_norm`, change
   `float* gd = (float*)c->gamma->data;` →
   `float* gd = c->gamma ? (float*)c->gamma->data->data : NULL;`
   (same for `bd`/beta).
2. Add `#include <math.h>` near the top of `ops.c`.

## Version references

All project version strings were bumped from `0.9.7.890` to `0.9.7.890e` in
`VERSION`, `CMakeLists.txt`, `pyproject.toml`, `bindings/python/setup.py`,
`Cargo.toml` (workspace), `net/distributed/Cargo.toml`, and `lib/rust/Cargo.toml`.
